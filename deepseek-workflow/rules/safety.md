# Safety Rules

执行者必须遵守这些规则：

1. 不允许执行 `git reset --hard`、`git clean`、`rm -rf`、强制 push。
2. 不允许改写任务目标；只能执行 `handoff.md` 中的任务。
3. 遇到不确定、依赖缺失、测试无法运行、权限不足时，写入 `BLOCKED` 并停止。
4. 不要泄露 API key、token、私钥或代理密码。
5. 改代码前先阅读相关文件，不能凭空修改。
6. 只改任务相关文件，避免无关重构。
7. 代码改完必须运行最小必要验证；无法运行时说明原因。
8. 所有执行动作和结果必须记录到 `execution.md`。
